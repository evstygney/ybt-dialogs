# Инструкция по редактированию упражнений

Эта инструкция предназначена для нетехнических пользователей: методистов, редакторов и владельцев продукта.

Здесь описано, как:

- менять тексты в уже существующих упражнениях;
- менять количество шагов внутри упражнения;
- добавлять новые упражнения;
- удалять упражнения;
- публиковать изменения на сайте.

## Где лежат упражнения

Все упражнения находятся в папке:

`data/scenarios/`

Каждый файл в этой папке — это одно отдельное упражнение.

Примеры:

- `data/scenarios/ya_self_care_map_01.json`
- `data/scenarios/biz_energy_budget_01.json`
- `data/scenarios/team_canvas_01.json`

Список упражнений, которые должны показываться на сайте, хранится в файле:

`data/manifest.json`

Проще говоря:

- в `data/scenarios/` лежит содержание упражнений;
- в `data/manifest.json` лежит список того, какие упражнения показывать в каждом модуле.

## Какие модули есть в проекте

В проекте три модуля:

- `Я`
- `Бизнес`
- `Команда`

Если вы добавляете новое упражнение, его нужно отнести к одному из этих трёх модулей.

## Как отредактировать текст существующего упражнения

### Что нужно открыть

1. Откройте папку `data/scenarios/`.
2. Найдите файл нужного упражнения.
3. Откройте его в редакторе текста или прямо на GitHub.

### Что можно менять без риска

Внутри файла можно спокойно менять:

- `title` — название упражнения;
- `description` — короткое описание;
- `objective` — цель упражнения;
- `estimated_minutes` — примерное время;
- `skills` — список навыков;
- `completion_feedback` — итоговый текст после завершения;
- `retry_tips` — подсказки для повторного прохождения;
- тексты шагов;
- тексты вариантов ответа;
- тексты последствий;
- тексты разборов после ответа.

### Что отвечает за шаги упражнения

У каждого упражнения есть блок:

`"steps": [ ... ]`

Внутри него перечислены шаги упражнения.

У каждого шага обычно есть:

- `scene_title` — короткий заголовок шага;
- `text` — основной текст ситуации;
- `options` — варианты ответов;
- `feedback.correct` — текст, если ответ правильный;
- `feedback.incorrect` — текст, если ответ неправильный.

## Как изменить количество шагов

Количество шагов регулируется внутри массива:

`"steps": [ ... ]`

### Чтобы добавить шаг

1. Скопируйте один из уже существующих шагов целиком.
2. Вставьте его внутрь массива `steps`.
3. Измените:
   - `step_id`
   - `scene_title`
   - `text`
   - варианты ответа
   - правильный вариант
   - тексты обратной связи

### Чтобы удалить шаг

1. Найдите ненужный шаг внутри `steps`.
2. Удалите его целиком от `{` до `}`.
3. Проверьте, чтобы после удаления не осталось лишней запятой.

## Как устроены варианты ответа

У каждого шага есть блок:

`"options": [ ... ]`

Каждый вариант ответа содержит:

- `id` — короткий идентификатор варианта, например `a`, `b`, `c`;
- `text` — текст варианта;
- `correct` — правильный ли ответ;
- `consequence` — что покажется пользователю сразу после выбора;
- `score` — сколько баллов даёт этот ответ;
- `ybt_delta` — вклад в профиль по трём осям;
- `tags` — служебные метки.

## Как отметить правильный ответ

В каждом шаге есть поле:

`"correct_option_id": "a"`

Это означает, какой вариант ответа считается правильным.

Важно:

- значение `correct_option_id` должно совпадать с `id` одного из вариантов;
- у правильного варианта обычно стоит `"correct": true`;
- у остальных вариантов обычно `"correct": false`.

Лучше, чтобы это совпадало всегда:

- `correct_option_id`
- и поле `correct` внутри соответствующего варианта

## Как менять баллы

У каждого варианта есть поле:

`"score": 10`

Можно менять баллы по смыслу упражнения:

- хороший ответ — больше баллов;
- нейтральный — мало баллов;
- слабый ответ — 0 или отрицательное значение.

## Как менять профиль по осям

У каждого варианта есть блок:

```json
"ybt_delta": {
  "self": 2,
  "business": 1,
  "team": 0
}
```

Это означает, как ответ влияет на три направления:

- `self` — Я
- `business` — Бизнес
- `team` — Команда

Можно менять эти значения по смыслу упражнения.

Пример:

- если ответ усиливает саморегуляцию — повышайте `self`;
- если ответ усиливает рабочую ясность — повышайте `business`;
- если ответ улучшает взаимодействие — повышайте `team`.

## Как добавить новое упражнение

Чтобы новое упражнение появилось на сайте, нужно сделать две вещи.

### Шаг 1. Создать новый файл упражнения

1. Откройте папку `data/scenarios/`.
2. Скопируйте похожее упражнение.
3. Переименуйте файл.

Пример имени файла:

`team_new_exercise_04.json`

4. Внутри файла обязательно поменяйте:

- `exercise_id`
- `module`
- `title`
- `description`
- `objective`
- `steps`

Важно:

- имя файла и `exercise_id` должны совпадать.

Пример:

- файл: `team_new_exercise_04.json`
- внутри: `"exercise_id": "team_new_exercise_04"`

### Шаг 2. Добавить упражнение в manifest

Откройте файл:

`data/manifest.json`

Найдите нужный модуль и добавьте новый `exercise_id` в список `exercise_ids`.

Например:

```json
{
  "name": "Команда",
  "exercise_ids": [
    "team_canvas_01",
    "team_circle_resp_02",
    "team_psych_safety_03",
    "team_new_exercise_04"
  ]
}
```

После этого упражнение появится на сайте.

## Как удалить упражнение

Чтобы упражнение исчезло с сайта:

1. Откройте `data/manifest.json`.
2. Удалите его `exercise_id` из списка `exercise_ids`.

После этого сайт перестанет его показывать.

Если нужно удалить его совсем из проекта, дополнительно удалите сам JSON-файл из `data/scenarios/`.

## Что лучше не менять без необходимости

Если нет особой причины, не трогайте:

- `engine.type`
- `engine.selection_mode`
- `engine.show_consequence_immediately`
- `engine.show_step_feedback`
- `engine.shuffle_options`

Эти поля отвечают за логику работы упражнения.

Их можно менять только если вы точно понимаете, зачем это нужно.

## Самые частые ошибки

### 1. Имя файла не совпадает с `exercise_id`

Неправильно:

- файл: `team_new_exercise_04.json`
- внутри: `"exercise_id": "team_new_exercise_05"`

Правильно:

- файл и `exercise_id` одинаковые.

### 2. Упражнение создано, но не добавлено в `manifest.json`

В этом случае файл существует, но на сайте упражнение не появится.

### 3. У шага указан неправильный `correct_option_id`

Например, написано:

`"correct_option_id": "d"`

Но варианта `d` в этом шаге нет.

### 4. После удаления шага осталась лишняя запятая

Это одна из самых частых причин поломки JSON.

### 5. Сломались кавычки

В JSON все текстовые значения должны быть в двойных кавычках:

`"title": "Название"`

## Как редактировать упражнения прямо на GitHub

Если не хочется работать с файлами на компьютере, можно редактировать проект прямо в браузере.

### Чтобы изменить существующее упражнение

1. Откройте репозиторий на GitHub.
2. Перейдите в папку `data/scenarios/`.
3. Откройте нужный файл.
4. Нажмите значок карандаша `Edit this file`.
5. Внесите изменения.
6. Нажмите `Commit changes`.

### Чтобы добавить новое упражнение

1. Откройте папку `data/scenarios/`.
2. Нажмите `Add file -> Create new file`.
3. Введите имя нового файла, например:
   `data/scenarios/team_new_exercise_04.json`
4. Вставьте содержимое нового упражнения.
5. Сохраните через `Commit changes`.
6. После этого отдельно отредактируйте `data/manifest.json` и добавьте туда новый `exercise_id`.

## Как опубликовать изменения на сайте

Если вы редактируете файлы прямо на GitHub:

- достаточно сохранить изменения через `Commit changes`.

После этого GitHub сам запустит обновление сайта.

Если вы редактируете проект на компьютере:

1. Сохраните изменения.
2. Загрузите их в репозиторий GitHub.
3. Подождите, пока GitHub обновит сайт.
